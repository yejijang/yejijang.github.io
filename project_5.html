Yumyum

Web 프로젝트 - 쌍용교육센터
2021.07.13 ~ 2021.08.06 (24일간)

⋆⁺₊⋆주제

배달 주문 웹 프로그램 - 사용자의 위치 기반으로 가게의 정보를 출력하고 주문할 수 있는 시스템

⋆⁺₊⋆개요

Yumyum은 JSP/Servlet를 이용하여 구현한 웹 프로젝트로, 배달 주문에 편리하게 사용할 수 있습니다.
사용자는 현재 위치 및 설정한 위치 기반으로 가게의 정보를 확인할 수 있으며 주문할 수 있고,
자신의 소비 내역으로 소비 패턴을 분석하여 취향을 쉽게 파악할 수 있게 구현하였습니다.

⋆⁺₊⋆담당업무

헤더 공동작업, 회원가입, 개인 정보 수정(회원 탈퇴), 주문내역 목록 출력 및 페이징, 주문내역 상세 보기, 주문내역 삭제,
가게 메뉴 선택(장바구니 추가), 장바구니 목록 출력, 장바구니 내역 삭제, 주문

⋆⁺₊⋆스킬

개발 환경: Windows, JDK1.8, Eclipse, eXERD, SQL Developer, Git
기술 키워드: Java, Oracle, JSP, Servlet, Ajax, API, JDBC, HTML5, CSS3, JavaScript, HighCharts API, 카카오 주소 API, Google geolocation API, 카카오 지도 API, JavaMail API..

⋆⁺₊⋆최종 후기

이 프로젝트는 JSP, Servlet 을 이용한 웹 프로젝트입니다.
기존에 존재하는 여러 배달 관련 웹, 앱 프로그램의 기능을 기반으로 구현하였으나,
사용하면서 불편했던 부분이나 추가적으로 필요하면 좋을 것 같은 기능들을 추가로 구현하였습니다.
일단, 장바구니 기능에서 여러 가게를 담지 못하고 한 가게의 메뉴만 담을 수 있는 부분에 있어
여러 가게의 메뉴를 장바구니에 담아 가격 및 정보를 확인할 수 있고, 가격 비교를 통해 해당하는 가게를 선택할 시 주문을 할 수 있도록 구현하였습니다.
또한 자신이 현재까지 결제한 총 금액을 확인할 수 있게 하였고,
소비 내역에 따른 소비 패턴을 파악해 그래프를 이용하여 소비자의 취향을 보기 편하게 구현하였습니다.

가게의 목록은 사용자의 현재 위치 설정(Google geolocation API, 카카오 지도 API)이나 위치 검색(카카오 주소 API) 방식을 이용하여
위치 기반 5km 반경의 가게를 보여주었습니다.
위치 계산은 vworld (지오코딩 API)를 사용하여 json 형식으로 리턴 받은 주소에서 위도, 경도를 파싱 한 후 가게의 좌표와 비교를 통해 진행하였습니다.

회원가입과 메뉴 선택 구현 시에는 비동기 방식으로 서버와 통신을 하기 위하여 Ajax를 사용하였습니다.
회원가입에서는 아이디, 닉네임 중복검사를 진행하여 모든 사용자의 정보와 비교한 후 아이디, 닉네임 사용 가능 여부를 구분하였습니다.
또한, 메뉴 선택은 선택한 메뉴에 따라 상세 정보 모달을 출력하여 장바구니에 추가할 수 있게 구현하였습니다.
기존에 Ajax로 모달 구현하는데 많은 두려움을 가지고 있었으나,
이번 계기로 인하여 Ajax의 구동 방식을 더욱 이해하는 데 도움이 되었고 json으로 받아와 데이터를 다듬어 출력하는데 적응할 수 있었습니다.

초기 구상 단계에서는 실제 결제를 구현하기 위해 결제 API를 사용하려 하였으나, 프로젝트 기한에 맞춰 성공하지 못하여 아쉬움이 있습니다.
현재는 주문을 하면 주문 내역 데이터가 데이터베이스에만 쌓이게 구현되어 있으나, 추후에는 결제 API를 붙여 실제 결제가 가능하도록 보완할 예정입니다.
