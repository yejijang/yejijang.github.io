교육센터 운영 프로그램

Oracle 프로젝트 - 쌍용교육센터
2021.05.24 ~ 2021.06.04 (12일간)

⋆⁺₊⋆주제

교육센터 운영 프로그램 - SQL, PL/SQL을 이용한 오라클 프로젝트

⋆⁺₊⋆개요

교육센터 운영 프로그램은 Oracle을 이용하여, 교육센터 운영에 필요한 SQL, PL/SQL을 구현한 프로젝입니다.
교육센터의 관리자는 과정 생성 및 강사, 교육생의 관리까지 센터 운영에 필요한 모든 관리를 할 수 있으며,
강사는 교육생 관리를, 교육생은 자신의 수강 과정에 따른 정보를 확인할 수 있습니다.

⋆⁺₊⋆담당업무

ERD, 테이블 생성, 프로시저 생성, 뷰 생성, 함수 생성, 트리거 생성, 인덱스 생성, 더미 데이터 생성, 기능 DML 구현

⋆⁺₊⋆스킬

개발 환경: Windows, eXERD, SQL Developer
기술 키워드: ERD, Oracle, PL/SQL, TRIGGER

⋆⁺₊⋆최종 후기

이 프로젝트는 Oracle을 이용한 프로젝트입니다.
대학교에서는 DB에 관련된 모든 걸 MySQL로 했었기에 오라클은 조금 생소하게 느껴졌으나,
DBMS의 비슷한 문법으로 학습하고 습득하는데 큰 어려움은 없었던 것 같습니다.

'교육센터 운영 프로그램'에 맞게 요구 사항을 분석하였고,
테이블과 각 속성을 도출해 내 eXERD 프로그램을 이용하여 ER diagram를 그렸습니다.
추가로 더미 데이터를 생성하여 TABLE, SEQUENCE를 비롯한 실제 프로그램이 구동될 때 더욱 편리하게 사용될 수 있도록,
FUNCTION, INDEX, VIEW, TRIGGER 또한 추가로 생성하였습니다.
이후에는 요구 분석서에 따라 각 기능에 해당하는 DML 작업을 진행하였습니다.

Oracle을 학습하면서 PL/SQL이라는 언어를 처음 접하게 되었는데,
비절차형인 SQL과 달리 데이터를 절차적으로 작업할 수 있다는 사실에 큰 이점을 느꼈습니다.
PROCEDURE을 이용할 시 변수를 사용하여 여러 DML문을 한 번에 처리할 수 있고, 조건문과 반복문 사용이 가능함에 있어 Java와 흐름이 비슷하다고 느꼈습니다.
하지만, 절차적 SQL문이 편리함에 따라 여러 제약 조건을 따져야 한다는 사실을 알게 되었습니다.

진행하는 동안, 구동 화면 없이 데이터베이스만 구현하는 프로젝트였기에 더욱 많은 어려움과 의문을 가졌던 것 같습니다.
요구 분석서에 따라 작업을 한다고 하여도 어떤 DML을 어디까지 작성해야 하는지 기준이 애매하였고,
하나의 select 문으로만 조회되지 않는 경우, 출력 결과를 만족시키기 위하여 다중 조인을 연속하여 6-7개 넘게 작성하는 데 어려움을 느꼈습니다.
또한 서버단에서 제약 조건을 걸어도 되는 부분을 PROCEDURE에서 모두 처리하는 것은 어느 정도 한계가 있다는 것을 깨달았습니다.
이번 프로젝트를 계기로 필요로 하는 요구를 분석하고 요구에 해당하는 데이터베이스 문을 작성하는데 더욱 능숙해졌고,
PL/SQL이 필요한 부분과 단순 SQL문을 사용하는 부분 또한 조금은 구분할 수 있게 된 것 같습니다.
평소에도 DBMS를 좋아하고 관심 가지던 분야였기에 즐겁게 임하였으며, 새롭게 알게 된 언어가 있어 뜻깊었던 프로젝트입니다.
